# Taste-Skill 开发路线图

## 整体规划

```
第1阶段（0-8周）    第2阶段（8-12周）    第3阶段（12-16周）   第4阶段（16+周）
MVP开发           功能完善           高级功能             持续优化
│                 │                  │                  │
v1.0             v1.5              v2.0               v2.1+
```

---

## 第 1 阶段（0-8 周）- MVP 开发

### 目标
- 完成核心的生产管理和设备管理功能
- 支持基本的生产看板展示
- 可进行第一个客户的试点

### 模块清单

#### 1.1 生产管理（4 周）

**第 1 周**：
- [ ] 订单管理基础功能
  - 订单创建、查询、编辑、删除
  - 订单状态流转（PLANNING → PRODUCING → COMPLETED）
  - 订单列表页（分页、搜索、筛选）

**第 2 周**：
- [ ] 工单管理
  - 从订单生成工单
  - 工单下发给班组
  - 工单进度追踪

**第 3 周**：
- [ ] 工人报工
  - 报工页面（手机端优先）
  - 报工数据提交和审核
  - 工单进度自动更新

**第 4 周**：
- [ ] 基础看板
  - 实时生产进度看板
  - 订单延期预警
  - 日报表（产量、不良率）

#### 1.2 设备管理（2 周）

**第 5 周**：
- [ ] 设备库管理
  - 设备信息录入
  - 设备分类和查询
  - 二维码生成

**第 6 周**：
- [ ] 点检和异常
  - 点检计划制定
  - 点检记录上传（手机端）
  - 异常自动上报
  - 维修单生成

#### 1.3 系统基础（2 周）

**第 7 周**：
- [ ] 用户和权限
  - 用户登录/注册
  - 基于角色的权限控制
  - 用户管理后台

**第 8 周**：
- [ ] 部署和测试
  - Docker 部署配置
  - 集成测试
  - 性能优化
  - 文档编写

### 技术实现

**后端**：
```
Java 17 + Spring Boot 3
├── api/
│   ├── OrderController
│   ├── WorkOrderController
│   ├── EquipmentController
│   └── AuthController
├── service/
│   ├── OrderService
│   ├── WorkOrderService
│   ├── EquipmentService
│   └── AuthService
├── repository/
│   ├── OrderRepository
│   ├── WorkOrderRepository
│   └── EquipmentRepository
└── entity/
    ├── Order
    ├── WorkOrder
    ├── Equipment
    └── User
```

**前端**：
```
Vue 3 + TypeScript + Vite
├── pages/
│   ├── OrderList.vue
│   ├── OrderDetail.vue
│   ├── WorkOrderList.vue
│   ├── Report.vue
│   ├── Dashboard.vue
│   ├── Equipment.vue
│   └── Inspection.vue
├── components/
│   ├── OrderForm.vue
│   ├── WorkOrderForm.vue
│   ├── ReportForm.vue
│   └── Navbar.vue
└── api/
    ├── order.ts
    ├── workOrder.ts
    ├── equipment.ts
    └── auth.ts
```

### 交付物
- 后端 API（可部署版本）
- 前端应用（Web 版）
- 手机端 H5（报工、点检）
- 数据库初始化脚本
- 部署文档和用户手册

## 里程碑和关键指标

### 第 1 阶段关键指标
- MVP 功能完成度 ≥ 90%
- 代码覆盖率 ≥ 70%
- API 响应时间 < 200ms
- 第 1 个客户成功上线

### 第 2 阶段关键指标
- 功能完整度 ≥ 95%
- 付费客户 ≥ 3 个
- 月收入 ≥ 5 万元
- 客户满意度 > 4/5

### 第 3 阶段关键指标
- 企业级功能完整
- 付费客户 ≥ 8 个
- 月收入 ≥ 15 万元
- 代理商 ≥ 2 个

### 第 4 阶段关键指标
- 生态合作伙伴 ≥ 5 个
- 月收入 ≥ 30 万元
- SaaS 用户 ≥ 50 个企业
- 市场占有率持续增长

## 风险和应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 需求变更频繁 | 高 | 中 | 定期需求评审，采用敏捷开发 |
| 技术难点（全文搜索、IoT） | 中 | 中 | 早期 POC，组件化设计 |
| 人员流失 | 中 | 高 | 竞争力薪酬，良好的工作环境 |
| 客户实施困难 | 中 | 中 | 完善的培训和文档 |
| 竞品威胁 | 低 | 高 | 持续创新，构建客户粘性 |
