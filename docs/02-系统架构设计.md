#### 3. 域层（Domain Layer）

**职责**：
- 定义实体和值对象
- 实现业务规则
- 不依赖外层框架

**关键实体**：
```java
// 订单实体
@Entity
@Table(name = "t_order")
public class Order {
    @Id
    private Long id;
    private String orderCode;
    private Long customerId;
    private String productName;
    private Integer quantity;
    private LocalDate dueDate;
    private OrderStatus status;  // 枚举
    private Priority priority;    // 枚举
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
    
    // 业务方法
    public void markAsProduction() {
        if (this.status != OrderStatus.PLANNING) {
            throw new BusinessException("只有待排产订单才能进入生产");
        }
        this.status = OrderStatus.PRODUCING;
    }
    
    public void markAsCompleted() {
        this.status = OrderStatus.COMPLETED;
    }
    
    public boolean isOverdue() {
        return LocalDate.now().isAfter(this.dueDate);
    }
}
```

```sql
CREATE TABLE t_order (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_code VARCHAR(50) UNIQUE NOT NULL,
    customer_id BIGINT NOT NULL,
    product_name VARCHAR(200) NOT NULL,
    quantity INT NOT NULL,
    unit VARCHAR(20),
```

## 安全架构

### 认证和授权

#### 认证流程
```
1. 用户登录（邮箱/用户名 + 密码）
2. 系统验证凭证，生成 JWT Token
3. 客户端保存 Token，后续请求包含 Authorization Header
4. 服务器验证 Token，提取用户信息
5. Token 过期后，客户端使用 Refresh Token 重新获取
```

#### JWT Token 结构
```
Header:   { "alg": "HS256", "typ": "JWT" }
Payload:  { 
    "sub": "user123",
    "email": "user@example.com",
    "roles": ["ADMIN", "SALES"],
    "exp": 1704067200
}
Signature: HMACSHA256(base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)
```

#### 权限控制

使用基于角色的权限控制（RBAC）：

```java
@RestController
@RequestMapping("/api/v1/orders")
public class OrderController {
    
    @PostMapping
    @RequireRole("PRODUCTION_MANAGER")  // 只有生产主管可以创建订单
    public ResponseEntity<OrderDTO> createOrder(@RequestBody CreateOrderRequest request) {
        // ...
    }
    
    @GetMapping("/{id}")
    @RequireRole({"ADMIN", "PRODUCTION_MANAGER", "WORKER"})  // 多角色
    public ResponseEntity<OrderDTO> getOrder(@PathVariable Long id) {
        // ...
    }
    
    @DeleteMapping("/{id}")
    @RequireRole("ADMIN")  // 只有管理员
    public ResponseEntity<Void> deleteOrder(@PathVariable Long id) {
        // ...
    }
}
```

### 数据安全

#### 密码加密
- 使用 BCrypt 算法
- 盐值由系统自动生成

#### 敏感数据加密
- 传输：HTTPS/TLS
- 存储：对敏感字段（如客户电话、邮箱）使用 AES 加密

#### 日志审计
```java
@Aspect
public class AuditAspect {
    // 记录所有数据修改操作
    // 包括：操作人、操作时间、操作类型、修改前后数据

    public Object audit(ProceedingJoinPoint joinPoint) throws Throwable {
        // 记录操作日志...
        return joinPoint.proceed();
    }
}
```

### 缓存策略

```
Cache-Aside 模式：
1. 读取数据时，先查 Redis
2. 如果缓存命中，返回数据
3. 如果缓存未命中，查数据库
4. 从数据库读出后，写入 Redis
5. 返回数据

缓存失效策略：
- 主动失效：更新/删除数据时，立即清除缓存
- 被动失效：设置 TTL，自动过期
```

### 消息队列异步处理

```
同步流程：
客户端请求 → 业务处理 → 数据库 → 发送通知 → 返回结果
                                    （慢）

异步流程：
客户端请求 → 业务处理 → 数据库 → 快速返回
                              ↓
                        消息队列（异步处理）
                              ↓
                        发送通知、生成报表等
```
