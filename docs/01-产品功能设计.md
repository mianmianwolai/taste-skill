# Taste-Skill 产品功能设计文档

## 文档概览

本文档详细描述 Taste-Skill 的完整功能设计，包括 5 个核心模块和跨模块的数据联动。

---

## 模块 1：生产管理（订单→排产→工单）

### 1.1 订单管理

#### 功能清单
- ✓ 创建订单（手动录入 / 导入 Excel）
- ✓ 订单基本信息（客户、产品、数量、交期）
- ✓ 订单状态跟踪（待排产、排产中、生产中、已完成、延期）
- ✓ 订单物料清单（BOM）
- ✓ 工艺路线绑定
- ✓ 订单进度查询
- ✓ 订单导出（PDF、Excel）

#### 数据字段
```
订单 (Order)
├── 订单号 (order_code)
├── 客户名称 (customer_name)
├── 产品名称 (product_name)
├── 订单数量 (quantity)
├── 单位 (unit)
├── 交期 (due_date)
├── 订单状态 (status: 待排产/排产中/生产中/已完成/延期)
├── 优先级 (priority: 低/中/高)
├── 备注 (remark)
├── 创建时间 (created_at)
└── 更新时间 (updated_at)
```

#### 用户旅程
1. 销售人员创建订单 → 系统生成唯一订单号
2. 生产主管查看待排产订单
3. 工单下发后，订单状态自动转为"生产中"
4. 工人报工完成后，订单状态转为"已完成"

---

### 1.2 排产管理

#### 功能清单
- ✓ 自动排产（基于优先级、交期、产能）
- ✓ 手动排产（拖拽界面）
- ✓ 产能规划
- ✓ 产线/班组管理
- ✓ 排产冲突预警
- ✓ 排产结果导出

#### 排产规则
```
优先级排序：高 > 中 > 低
交期排序：最早交期优先
产能约束：不超过产线日产能
人力约束：根据班组规模调整
```

#### 排产界面逻辑
- 左侧：待排产订单列表
- 中间：甘特图（产线 × 时间轴）
- 右侧：产能/资源面板
- 操作：拖拽订单到产线，系统计算工期

---

### 1.3 工单管理

#### 功能清单
- ✓ 工单自动生成（从订单排产）
- ✓ 工单详情（工艺、物料、设备、预期时间）
- ✓ 工单下发给班组
- ✓ 工单状态管理（待生产、生产中、已完成、暂停、取消）
- ✓ 工单追踪
- ✓ 工单历史查询

#### 数据字段
```
工单 (WorkOrder)
├── 工单号 (work_order_code)
├── 订单号 (order_id)
├── 产品名称 (product_name)
├── 工单数量 (quantity)
├── 工艺路线 (process_id)
├── 指派班组 (team_id)
├── 预期开始时间 (expected_start)
├── 预期完成时间 (expected_end)
├── 实际开始时间 (actual_start)
├── 实际完成时间 (actual_end)
├── 状态 (status)
├── 完成进度 (progress_percent: 0-100)
└── 备注 (remark)
```

#### 工单流转
```
待生产 → 生产中 → 已完成
   ↓         ↓
 取消    暂停 → 继续
```

---

### 1.4 工人报工

#### 功能清单
- ✓ 工人打卡（扫工单二维码或手动选择）
- ✓ 本次报工数量输入
- ✓ 不良品数量记录
- ✓ 工时计算（系统自动计算耗时）
- ✓ 班长审核报工
- ✓ 报工历史查询

#### 报工流程
```
1. 工人扫工单条码 → 自动关联工单
2. 选择报工类型（正常/返工/废品）
3. 输入数量、耗时
4. 添加备注（可选）
5. 提交
6. 班长手机端审核（可选模式）
7. 系统自动更新工单进度
```

#### 数据字段
```
报工记录 (Report)
├── 报工号 (report_code)
├── 工单号 (work_order_id)
├── 工人 ID (worker_id)
├── 报工数量 (quantity)
├── 不良品数量 (defect_quantity)
├── 耗时（分钟）(duration_minutes)
├── 报工时间 (report_time)
├── 审核状态 (status: 待审/已审/驳回)
├── 审核人 (reviewer_id)
└── 备注 (remark)
```

---

### 1.5 生产看板

#### 功能清单
- ✓ 实时产线看板（各产线当前工单进度）
- ✓ 订单延期预警（红、黄、绿三色）
- ✓ 班组产能对标
- ✓ 日周月报表
- ✓ 生产效率指标（OEE、产能利用率）
- ✓ 瓶颈工序分析

#### 看板指标
| 指标 | 计算方式 | 目标 |
|------|---------|------|
| 产能利用率 | 实际产出 / 计划产出 | > 85% |
| 按期交付率 | 按期完成订单数 / 总订单数 | > 90% |
| 一次良率 | 无返工产品数 / 总产品数 | > 95% |
| 工序转移时间 | 前工序完成 - 后工序开始 | < 2 小时 |

### 2.2 点检管理

#### 功能清单
- ✓ 点检计划制定（日/周/月/季度）
- ✓ 点检清单模板
- ✓ 移动端点检打卡
- ✓ 点检项目记录（温度、噪音、振动等）
- ✓ 异常自动上报
- ✓ 点检历史查询
- ✓ 点检任务提醒

#### 点检流程
```
1. 管理员制定点检计划（哪些设备、什么频率）
2. 设备员手机扫设备二维码
3. 按照点检清单逐项记录
4. 如发现异常，标记为"异常"
5. 系统自动生成异常上报单
6. 异常单流转到维修班组
```

#### 数据字段
```
点检计划 (InspectionPlan)
├── 计划号 (plan_id)
├── 设备号 (equipment_id)
├── 点检频率 (frequency: 日/周/月/季/年)
├── 点检项目 (inspection_items: 数组)
├── 创建时间 (created_at)
└── 执行情况 (execution_status)

点检记录 (InspectionRecord)
├── 记录号 (record_id)
├── 计划号 (plan_id)
├── 点检员 (inspector_id)
├── 点检时间 (inspection_time)
├── 各项记录 (item_records: {项目名: 值, 是否异常: true/false})
├── 照片 (photos: URL 数组)
└── 备注 (remark)
```

---

### 2.3 异常上报

#### 功能清单
- ✓ 异常自动上报（点检发现）
- ✓ 手动上报（设备员/工人）
- ✓ 异常分类（电气故障/机械故障/液压故障等）
- ✓ 异常等级（紧急/高/中/低）
- ✓ 异常描述和现场照片
- ✓ 异常推送提醒（微信、钉钉、短信）

#### 异常等级
| 等级 | 特征 | 响应时间 |
|------|------|---------|
| 紧急 | 导致停线 | 5 分钟内响应 |
| 高 | 严重故障 | 15 分钟内响应 |
| 中 | 轻微故障 | 1 小时内响应 |
| 低 | 预防性维护 | 24 小时内响应 |

#### 功能清单

```
维修单 (MaintenanceOrder)

├── 异常上报单号 (exception_report_id)
├── 故障描述 (fault_description)
├── 维修方案 (repair_plan)
├── 指派维修工 (assigned_worker_id)
├── 预期维修时间 (expected_duration)
├── 实际维修时间 (actual_duration)
├── 维修零件 (spare_parts: 数组)

```

### 2.5 设备统计看板

#### 功能清单
- ✓ 设备故障率统计
- ✓ 设备停线时间
- ✓ 设备稼动率（MTBF, MTTR）
- ✓ 预防性维护完成率
- ✓ 维修成本趋势
- ✓ 频繁故障设备预警

#### 关键指标
```
设备稼动率 = (日历时间 - 故障停机时间) / 日历时间
MTBF = 总正常运行时间 / 故障次数（平均故障间隔时间）
MTTR = 总维修时间 / 故障次数（平均维修时间）
```

### 3.3 不良品管理

#### 功能清单
- ✓ 不良品分类

- ✓ 返工/报废决策
- ✓ 不良品处理流程

#### 不良品处理流程
```
发现不合格 → 隔离 → 分析原因 → 决策（返工/报废/特采）
                          ↓
                    返工：重新加工
                    报废：报废处理
                    特采：特别批准
```

#### 数据字段
```
不良品 (DefectProduct)
├── 不良品号 (defect_id)
├── 检验记录号 (quality_record_id)
├── 产品代码 (product_code)
├── 不良分类 (defect_category: 尺寸/外观/功能等)
├── 不良等级 (severity: 严重/一般/轻微)
├── 处理方式 (handling: 返工/报废/特采)
├── 处理人 (handler_id)
├── 处理日期 (handled_date)
└── 备注 (remark)
```

### 3.4 批次追溯

#### 功能清单

```
原材料 (供应商)

    ↓
出库 (发货日期、客户、物流号)
```

## 跨模块数据联动

### 联动 1：订单 → 工单 → 报工 → 质检

```
1. 销售创建订单
2. 生产排产生成工单
3. 工人扫工单条码报工
4. 系统自动关联质检清单
5. 质检员检验
6. 追溯系统记录整个过程
```

### 联动 2：设备异常 → 维修 → 工单延期预警

```
1. 设备故障上报
2. 自动生成维修单
3. 如果预计停机 > 1 小时，触发工单延期预警
4. 生产主管收到通知，重新排产
```

### 联动 3：质检不合格 → 返工工单 → 成本统计

```
1. 质检发现不合格
2. 如选择返工，系统自动生成返工工单
3. 返工成本自动累加到产品成本
4. 销售看板显示因质量而产生的额外成本
```

### 联动 4：销售报价 → 订单 → 成本对标

```
1. 销售报价时，系统根据历史数据估算成本
2. 订单生成后，真实成本在生产中逐步累积
3. 订单完成后，自动对标报价成本和实际成本
4. 生成偏差分析报告供销售和生产优化
```
